# ポートフォリオ用サイトを作りました
https://nanbu-kyousitsu-dammy.netlify.app/  
## 概要
私moppi-rgbが所属する事業所の関連施設である「[放課後デイサービス 南部教室](https://www.sougoufukushi.com/)」のWebサイトをリニューアルするという想定で制作したものです。  
学習用としての制作物であり、実際に使用されるものではありません。
## 制作背景
Web制作を学習するにあたり、所属事業所のスタッフさんから上記のコンセプトのページを作ることをご提案いただきまして、企画からデプロイまでほぼ全ての過程を担当・開発しました。  
実際に使用するサイトではないため、トップページとフォームページのみを制作しています。
### 学習目標
*1.Web制作のプロセスを体系的に学ぶ*   
*2.学んだことのない技術を使用してみる*  
*3.実際に動かせるお問い合わせフォームを作る*

## 制作の流れ
### 1.企画・サイト設計
まずは既存サイトをよりモダンなデザイン・レイアウトで作り直すことを重視しました。  
事業所スタッフさんをクライアントと見立てて聴き取りを行い、ペルソナを小さいお子さんのいる親御さんに設定。  
お子さんのことで不安なことがある中でも安心して情報収集できるように、余白や色味・情報量などに気を付けたデザインを目指すことにしました。  
設計については既存サイトの情報を複数ページに分けることで、1ページの文章量を削減して全体的にゆったりした作りになるようにしています。
### 2.デザイン
上記の設計を基にFigmaでカンプを作成。スタッフさんに何度か修正をいただきながらブラッシュアップしていきました。  
本来であればプロトタイプ機能を用いて動作まで設定すべきだと思いますが、今回レイアウトを作るのにとても時間がかかってしまったため静的なデザイン作成のみ行っています。  
サイト中のリード文については既存サイトの文章を基に、スタッフさんにリライトしていただきました。他文章は同じく既存サイトを基に自分でリライトしています。
### 3.コーディング
ビルドにViteを使用、Reactで実装しています。  
デザインをしっかり作ったのでコーディングは簡単！と思っていたのですが、使ったことのないReactを使用したためにこちらもとんでもなく時間がかかってしまいました…。
レスポンシブにも対応させており、どのデバイスで見たときもなるべく破綻がないようにしています。
## 反省点・課題点
本来Astroを使用してサイト制作をする予定でした。しかしデザインから作るということで想像より遥かに制作に時間がかかってしまい、見切り発車で日本語チュートリアルが充実しているReactを使用することにしました。  
そのため理解が及ばないまま実装を進めてしまい、途中からどうコードを書くべきか分からなくなってしまったことで、後半はかなりCopilotに頼っての開発になりました(具体的にはReactの他にreact-hook-form, Zod, Motion, Netlifyへのデプロイ周りなどをかなりCopilot任せにしています)。  
理解していない技術をバイブコーディングで実装するのはとても危険ですね…。今回はとにかくリリースしたかったので強引に進めましたが、今後はそういったことがないようにきちんと技術内容を学習してから開発に臨むことを心がけたいです。
